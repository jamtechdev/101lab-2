Requirement 2 — why it was replaced
The react-native-ble-peripheral npm package has a fundamental bug: its Android onCharacteristicWriteRequest callback creates the event map but never calls sendEvent() to JS — so your server would never receive any command. You'd get an advertising peripheral that is completely deaf to writes.

The custom BlePeripheralModule.java (in android/app/src/main/java/com/bledemo/) does everything the npm package was supposed to do, correctly:

Proper GATT server with CCCD descriptor for notifications
Write events emitted to JS as BlePeripheralWriteRequest
Base64 in/out matching ble-plx
Handles all connected devices for notify broadcasts
One real-world caveat
Not all Android phones support peripheral/advertiser mode — it requires hardware support for BLE LE peripheral role (BluetoothAdapter.isMultipleAdvertisementSupported()). The native module will reject the start() promise with ADVERTISER_UNAVAILABLE if the hardware doesn't support it. Most phones from 2018+ support it.
